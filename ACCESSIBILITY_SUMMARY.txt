================================================================================
CRITICAL ISSUES (MUST FIX)
================================================================================

1. NO SKIP NAVIGATION LINK
   File:     src/app/layout.tsx, src/app/(dashboard)/layout.tsx
   WCAG:     2.4.1 Bypass Blocks (Level A)
   Impact:   Screen reader users cannot skip 30+ navigation items
   Fix:      Add hidden skip link at start of body

2. FORM ERRORS NOT ASSOCIATED WITH INPUTS
   File:     src/app/(auth)/login/page.tsx:78-80
            src/app/(auth)/signup/page.tsx:113-115
   WCAG:     3.3.1 Error Identification (Level A)
   Impact:   Users don't understand which field has error
   Fix:      Add role="alert" and aria-describedby to link error to input

3. COLOR CONTRAST FAILURES
   File:     src/components/ui/progress.tsx:17
            src/components/compliance/data-completeness-card.tsx:28
            src/components/compliance/portfolio-dashboard-client.tsx (multiple)
   WCAG:     1.4.3 Contrast (Minimum) AA 4.5:1
   Impact:   Low vision users cannot read text
   Fix:      Test with contrast checker; adjust opacity/color values

4. CHARTS WITHOUT DATA TABLES
   File:     src/components/compliance/fuel-breakdown-chart.tsx:63
            src/components/compliance/monthly-emissions-chart.tsx:90
            src/components/compliance/emissions-trend-chart.tsx
            src/components/readings/reading-chart.tsx
   WCAG:     1.1.1 Non-text Content (Level A)
   Impact:   Cannot access chart data without visual interpretation
   Fix:      Add <details> element with table of values

5. MISSING LANDMARK REGIONS
   File:     src/app/(auth)/login/page.tsx
            src/app/(auth)/signup/page.tsx
   WCAG:     1.3.1 Info and Relationships (Level A)
   Impact:   Cannot navigate page structure with assistive technology
   Fix:      Add <header>, <main>, <footer> semantic landmarks

================================================================================
MAJOR ISSUES (PREVENT AA)
================================================================================

6. MISSING ICON ALT TEXT
   Files:    Throughout codebase (40+ icons)
   Examples: src/app/(dashboard)/buildings/[id]/page.tsx:88, 171, 181
   WCAG:     1.1.1 Non-text Content (Level A)
   Impact:   Decorative icons confuse or block screen readers
   Fix:      Add aria-label or aria-hidden="true" to all icons

7. BROKEN HEADING HIERARCHY
   Files:    Multiple pages and components
   Examples: src/app/(dashboard)/buildings/[id]/page.tsx:59 (h2 without h1)
            src/components/compliance/what-if-calculator.tsx:132 (h4 without h2/h3)
   WCAG:     1.3.1 Info and Relationships (Level A)
   Impact:   Cannot navigate pages with headings
   Fix:      Ensure every page has h1, then h2, h3 in sequence

8. CUSTOM CONTROLS WITHOUT ARIA
   File:     src/components/compliance/what-if-calculator.tsx:102-122
   WCAG:     4.1.2 Name, Role, Value (Level A)
   Impact:   Cannot understand slider/input controls
   Fix:      Add aria-label, aria-valuemin, aria-valuemax, aria-valuenow

9. TABLE HEADERS WITHOUT SCOPE
   Files:    Multiple table components
   Example:  src/components/emissions-breakdown-table.tsx:47-52
   WCAG:     1.3.1 Info and Relationships (Level A)
   Impact:   Cannot understand table structure
   Fix:      Add scope="col" to all <th> elements

10. POOR LINK FOCUS INDICATORS
    Files:    src/components/ui/breadcrumb.tsx:27
             src/app/(dashboard)/buildings/[id]/page.tsx:168-194
    WCAG:     2.4.7 Focus Visible (Level AA)
    Impact:   Keyboard users cannot see where they are
    Fix:      Add :focus-visible CSS or outline classes

================================================================================
MINOR ISSUES (IMPROVE UX)
================================================================================

11. MISSING ARIA-LIVE REGIONS
    Files:    src/components/compliance/what-if-calculator.tsx (results)
             src/components/compliance/portfolio-dashboard-client.tsx (pagination)
    WCAG:     4.1.3 Status Messages (Level AAA)
    Impact:   Dynamic content changes announced
    Fix:      Add aria-live="polite" to dynamic regions

12. SEMANTIC LINK/BUTTON CONFUSION
    File:     src/app/(dashboard)/buildings/[id]/page.tsx:168-194
    WCAG:     1.3.1 Semantic Structure
    Impact:   Card links confuse semantic structure
    Fix:      Make only text/button the link, not entire card

13. BREADCRUMB ACCESSIBILITY
    File:     src/components/ui/breadcrumb.tsx
    WCAG:     1.3.1, 2.4.7
    Impact:   Navigation not fully accessible
    Fix:      Add aria-label to nav, ensure focus visible

14. FORM ELEMENT LABELING GAPS
    Files:    Various forms missing aria-required, aria-invalid
    WCAG:     1.3.1, 3.3.2
    Impact:   Form state not fully conveyed
    Fix:      Ensure all form elements have proper attributes
